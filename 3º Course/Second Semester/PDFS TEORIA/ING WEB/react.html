1. OBJETIVOS

  • Preparar nuestro entorno de trabajo para desarrollar aplicaciones con React disfrutando de algunas de las "ayudas" que nos proporciona VS Code.
  • Conocer algunos conceptos básicos de JavaScript que nos serán útiles para trabajar con React.
  • Conocer los conceptos básicos de React y empezar a trabajar con ellos.
  • Realizar un ejercicio práctico para poner en práctica lo aprendido.

2. PREPARACIÓN DEL ENTORNO

Antes de empezar, vamos a preparar nuestro entorno de trabajo. No te saltes estos pasos... ¡son importantes y nos ahorrarán mucho tiempo más adelante!

2.1. VS Code

Quizás tengas otro editor de código favorito, o le hayas cogido gusto a editar con kate o Geany... Nos han sido muy útiles para aprender, pero para trabajar con React, VS Code es una herramienta muy potente y que nos va a facilitar mucho la vida. Descarga e instala VS Code desde https://code.visualstudio.com/download. En xubuntu, descarga el archivo .deb y ejecuta el siguiente comando en la terminal:

sudo dpkg -i <ruta_del_fichero_.deb>

2.2. Deshazte del copiloto

Recuerda que nuestro objetivo no es programar una aplicación, sino aprender a programarla.

GitHub Copilot (o equivalentes) son herramientas muy potentes, pero para aprender a programar, es mejor hacerlo a mano. Si tienes GitHub Copilot instalado en VS Code, desactívalo. Pulsa Ctrl+Shift+P y escribe GitHub Copilot: Dissable Copilot completions para desactivarlo.

2.3. Node.js y NPM

Node.js es un entorno de ejecución para JavaScript que permite ejecutar código JS fuera del navegador. Esto es crucial para el desarrollo de aplicaciones web por varias razones:

  • Servidor de desarrollo: Node.js es utilizado para correr el servidor de desarrollo local mientras construyes tu aplicación React. Este servidor te permite ver cambios en tiempo real en el navegador a medida que desarrollas tu código. Significa esto que necesitaremos un servidor corriendo node para ejecutar/desplegar nuestro proyecto? Respuesta: NO! Solo durante el desarrollo. Podremos exportar nuestras páginas a contenido estático!
  • Entorno de ejecución para herramientas de construcción: Muchas herramientas necesarias para desarrollar, probar y depurar aplicaciones React se ejecutan sobre Node.js. Estas incluyen webpack, Babel, y varias otras que transforman el código moderno de JavaScript y React para que sea compatible con múltiples navegadores.

NPM es el gestor de paquetes de Node.js. Es una herramienta que te permite instalar y gestionar paquetes de Node.js. Los paquetes son módulos de código que puedes usar en tu aplicación.

Lo mejor es que instalemos ambos mediante un gestor de versiones como NVM (Node Version Manager). Así podremos cambiar de versión de Node.js y NPM fácilmente en caso de que sea necesario.

  1. Instalamos nuestra versión correspondiente de NVM
    • Linux o OSX: https://github.com/nvm-sh/nvm
      • curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
    • Windows: https://github.com/coreybutler/nvm-windows (No te bajes el código fuente, no queremos compilarlo de 0. Descarga el instalador desde Releases e instala NVM.)
  2. Cerramos y volvemos a abrir la terminal
  3. Instalamos la versión de Node.js y NPM que necesitamos
    • nvm install latest
  4. Indicamos que queremos emplear la versión que acabamos de instalar. (en mi casi la 21.7.3)
    • nvm use 21.7.3
  5. Comprobamos que todo ha ido bien
    • nvm list
    • node -v
    • npm -v

2.3.1. Inicialización de un proyecto de react

Ahora que tenemos node.js y npm operativos, vamos a crear un proyecto de React. Ve a la carpeta donde quieras crear tu proyecto y ejecuta el siguiente comando para crear tu proyecto dentro de dicha carpeta. El punto al final indica que queremos crear el proyecto en la carpeta actual:

npx create-react-app .

2.4. VS Code Plugins

Instalemos algunos plugins esenciales.

2.4.1. Prettier - Code Formatter

Prettier es una herramienta de formateo de código que nos ayudará a mantener nuestro código limpio y ordenado.

  1. Instalalo desde el marketplace de VS Code.
  2. Abre el archivo de configuración de VS Code pulsando Ctrl+,.
  3. Define prettier como default formatter.
  4. Activa la opción de formatear el código al guardar "Editor: Format on Save".

2.4.2. ESLint y errorLens

ESLint es una herramienta de análisis de código estático para identificar problemas en nuestro código JavaScript. errorLens es un plugin que nos permite ver los errores de ESLint directamente en el código.

  1. Mediante la terminal, instala ESLint de forma global.

    npm install -g eslint

  2. Instala el plugin de ESLint en VS Code.

  3. Ubicate en la carpeta de tu proyecto y genera el fichero de configruación de ESLint.

    eslint --init

  4. Responde a las preguntas que te haga el asistente.

    • √ How would you like to use ESLint? · To check syntax and find problems
    • √ What type of modules does your project use? · JavaScript module (import/export)
    • √ Which framework does your project use? · react
    • √ Does your project use TypeScript? · No, only javascript
    • √ Where does your code run? · browser
    • The config that you've selected requires the following dependencies:
      • eslint, globals, @eslint/js, eslint-plugin-react
      • √ Would you like to install them now? · No / Yes
      • √ Which package manager do you want to use? · npm

  5. Comprueba que:

    • Se ha creado el fichero .eslintrc.json con configuración de ESLint. Si en su lugar tienes un fichero eslint.config.mjs, puede que se te haya activado una configuracion experimental y tengas que hacer Ctrl+, -> Buscar “eslint.experimental” y activar “Experimental: Use Flat Config”.

    • Se ha modificado tu fichero package.json incluyendo información sobre las nuevas dependencias

    • Al trabajar con un fichero .js, si cometemos errores (por ejemplo declarar variables nunca utilizadas) nos sale una advertencia al respecto.

    • Es posible que ESLint se queje de no estemos importando REACT en algunos componentes. Esto se debe a que desde react 17 ya no es necesario hacerlo así. Debemos informar a ESlint de que no se trata de un error. Modificamos el fichero eslint.config.mjs añadiendo unas reglas extra:

      export default [
  { languageOptions: { globals: globals.browser } },
  pluginJs.configs.recommended,
  pluginReactConfig,
  {
    rules: {
      "react/react-in-jsx-scope": "off",
      "react/jsx-uses-react": "off",
      "react/prop-types": "off",
    },
  },
];

  6. Esto todavía puede mejorar...Instala mediante plugins “error lens”. Mejor, verdad?

2.4.3. Gitlens

Gitlens es una herramienta que nos permite ver información de git directamente en el código. Nos permite ver quién ha hecho un cambio, cuándo se hizo, y mucho más. Bastante util para cuando trabajamos en equipo.

  1. Instalalo desde el marketplace de VS Code.
  2. Cuando trabajes con un proyecto que forme parte de un repositorio de git, verás en cada linea de código quién la modificó, cuándo y con qué commit.
  3. También se resaltarán los ficheros pendientes de commitear

2.4.4. Live Share

Live Share es una herramienta que nos permite colaborar en tiempo real con otros desarrolladores. Nos permite compartir nuestro entorno de desarrollo con otros, permitiéndoles ver y editar nuestro código.

  1. Busca e instala el plugin “Live Share”
  2. En la barra inferior de VS code aparecerá el botón “Live Share”. Púlsado e inicia sesión con github.
  3. Comparte el enlace con quien desees, y colaborad. Como en figma, puedes chatear, centrar la atencion en un compañero o solicitar que te la presten. También compartir terminal o servidor TCP.

2.4.5. Quokka

Quokka es una herramienta que nos permite ejecutar código JavaScript en tiempo real. Nos permite ver el resultado de nuestro código en tiempo real, sin necesidad de ejecutarlo en un navegador. Util para trastear con algunos conceptos de JS.

  1. Busca e installa Quokka.js mediante plugins
  2. Regresa a tu fichero JS
  3. Pulsa CTRL+SHIFT+P y ejecuta el comando “Quokka.js: Toggle (Start/Stop) on Current File
  4. Observa como aparecen en vivo los valores de la ejecución de cada linea y cómo junto al nº de linea se indica sí estaría siendo ejecutada o no.

3. ALGUNOS CONOCIMIENTOS PREVIOS DE JS

Para trabajar con React, es necesario tener algunos conocimientos previos de JavaScript. A continuicación, vamos a repasar algunos conceptos básicos que nos serán útiles.

3.1. Arrow Functions

https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Functions/Arrow_functions

Una arrow function es una sintaxis concisa para escribir funciones en JavaScript que no tiene su propio this, arguments, super, o new.target y es típicamente utilizada para funciones cortas y anónimas.

const elementos = ["manzana", "naranja", "kiwi"];
// Ejemplo con function tradicional
elementos.map(function (elemento) {
  return "<li>elemento</li>";
});

//Ejemplo con arrow function
elementos.map((elemento) => "<li>elemento</li>");

Observa que en la arrow function no es necesario escribir la palabra reservada function, y que si la función solo tiene una línea de código, no es necesario escribir las llaves {}. Además, si la función solo tiene un argumento, no es necesario escribir los paréntesis ().

Un par de ejemplos más:

// Arrow function con dos parámetros
const miFunc = (a, b) => a + b * b;
console.log(miFunc(1, 2));

// Arrow function anónima como handler de evento (Las utilizaremos MUCHO!)
<button
  onClick={() => {
    console.log("HelloWorld");
  }}
></button>;

3.2. Operadores ternarios y operadores lógicos

  • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Conditional_operator
  • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Logical_AND

Nos hemos acostumbrado a utilizar if/else para controlar el flujo de nuestro código. Sin embargo, en JavaScript, podemos utilizar operadores ternarios y operadores lógicos para hacerlo de una forma más concisa.

let myEdad;

// If tradicional
myEdad = 30;
if (myEdad > 18) {
  console.log("adulto");
} else {
  console.log("joven");
}

myEdad = 10;
let rangoEdad;

// Aprovechando los operadores lógicos
rangoEdad = myEdad > 18 && "adulto";
console.log(rangoEdad);
rangoEdad = (myEdad > 18 && "adulto") || "joven";
console.log(rangoEdad);

// Operador ternario
rangoEdad = myEdad > 18 ? "adulto" : "joven";
console.log(rangoEdad);

// Será util en react al devolver componentes condicionalmente
const Component = () => {
  return myEdad > 18 ? <h1>Adulto</h1> : <h1>Joven</h1>;
};

3.3. Deestructuring

https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Destructuring_assignment

El deep destructuring es una técnica que nos permite extraer múltiples valores de un objeto o array y asignarlos a variables de golpe.

const person = {
  nombre: "Inigo",
  edad: 30,
  sabeJavascript: false,
};

// Método tradicional de extraer propiedades de un objeto
person;
let userNombre = person.nombre;
let userEdad = person.edad;
let userSabeJavascript = person.sabeJavascript;
userNombre;
userEdad;
userSabeJavascript;

// Método mediante destructuring, en una sola linea. (Las vairables y propiedades deben tener el mismo nombre)
const { nombre, edad, sabeJavascript } = person;
nombre;
edad;
sabeJavascript;

Fíjate que en el destructuring, las variables que se crean deben tener el mismo nombre que las propiedades del objeto. Si queremos asignar un nombre diferente a la variable, podemos hacerlo de la siguiente forma:

const {
  nombre: myNombre,
  edad: myEdad,
  sabeJavascript: mySabeJavascript,
} = person;
myNombre;
myEdad;
mySabeJavascript;

El deestructuring nos va a ser muy util en React para extraer propiedades de un objeto y pasarlas como props a un componente. Por ejemplo:

const Component = (props) => {
  const { nombre, edad, sabeJavascript } = props;
  return (
    <div>
      <h1>{nombre}</h1>
      <h2>{edad}</h2>
      <h3>{sabeJavascript ? "Sí" : "No"}</h3>
    </div>
  );
};

3.4. Spread Operator

https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Spread_syntax

El spread operator es una sintaxis que nos permite expandir elementos de un array o propiedades de un objeto. Nos va a ser muy útil para trabajar con arrays y objetos de una forma más cómoda.

Supongamos que tenemos dos arrays y queremos unificarlos. Tradicionalmente haríamos:

const array1 = [1, 2, 3];
const array2 = [4, 5, 6];
const array3 = array1.concat(array2);
console.log(array3);

Con el spread operator podemos hacer

const array1 = [1, 2, 3];
const array2 = [4, 5, 6];
const array3 = [...array1, ...array2];
console.log(array3);

Propongamos un ejemplo mas cercano a nuestro futuro uso en react.

// Supongamos que tenemos un componente que recibe ciertas propiedades
function Saludo({ firstName, lastName }) {
  return (
    <h1>
      HolaHolita, {firstName} {lastName}!
    </h1>
  );
}

// Si ahora quisese crear un componente de ese tipo con estas propiedades...
const props = {
  firstName: "Inigo",
  lastName: "Ezcurdia",
  edad: 30,
  sabeJavascript: true,
  sabeReact: false,
  sabeAngular: false,
};
const MyComponent = (
  <Greeting
    firstName={props.firstName}
    lastName={props.lastName}
    sabeJavascript={props.sabeJavascript}
    sabeReact={props.sabeReact}
    sabeAngular={props.sabeAngular}
  />
);

// Vaya peñazo escribir todas las propiedades no?. Con el spread operator lo hacemos en una sola linea
const MyComponent = <Greeting {...props} />;

3.5. Find

https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/find

La función find() devuelve el primer elemento de un array que cumple con una condición dada. Si no encuentra ningún elemento, devuelve undefined.

const elementos = ["manzana", "naranja", "kiwi"];
const elemento = elementos.find((elemento) => elemento === "naranja");
console.log(elemento);

3.6. Map

https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/map

Vamos a utilizar MUCHO la función map. La función map() crea un nuevo array con los resultados de la llamada a la función indicada aplicados a cada uno de sus elementos. Ejemplo, pasar a minúsculas todos los nombres de una lista.

// Método tradicional
listaNombres = ["Inigo", "Roberto", "Juan", "Iñaki"];
let listaNombresLower = [];
listaNombres.forEach((nombre) => {
  listaNombresLower.push(nombre.toLowerCase());
});
console.log(listaNombresLower);

// Mediante map
listaNombres = ["Inigo", "Roberto", "Juan", "Iñaki"];
listaNombresLower = listaNombres.map((nombre) => nombre.toLowerCase());
console.log(listaNombresLower);

Una vez más, veamos un ejemplo más cercano a nuestro futuro uso en React.

// Tendremos un listado de personas y por cada persona querremos crear un elemento <li>
listaNombres.map((nombre) => return <li>{nombre}</li>);

3.7. Filter

https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/filter

La función filter() crea un nuevo array con todos los elementos que cumplan con la condición dada. Nos va a ser especialmente util para filtrar elementos de un array.

listaNombres = ["Inigo", "Roberto", "Juan", "Iñaki"];
let listaNombresI = [];

// Filtrado tradicional (pero con arrow functions!)
listaNombres.forEach((nombre) => {
  if (nombre.startsWith("I")) {
    listaNombresI.push(nombre);
  }
});
console.log(listaNombresI);

// Mediante filter (y arrow functions, claro!)
listaNombres = ["Inigo", "Roberto", "Juan", "Iñaki"];
listaNombresI = listaNombres.filter((nombre) => nombre.startsWith("I"));
console.log(listaNombresI);

3.8. Promesas, async/await

  • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Using_promises
  • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/async_function
  • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/await

En javascript, las promesas son objetos que representan la terminación o el fracaso de una operación asíncrona.

Las promesas tienen tres estados, pendiente, cumplida y rechazada. Una vez que una promesa se resuelve, se puede acceder a su valor mediante el método then(), o manejar un error mediante el método catch().

// Ejemplo de promesa
const miPromesa = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("¡Éxito!");
  }, 1000);
});
miPromesa
  .then((value) => console.log(value))
  .catch((error) => console.log(error));

async se utiliza para declarar funciones asíncronas. Una función asíncrona es una función que devuelve una promesa. await se utiliza para esperar a que una promesa se resuelva.

async function fetchData() {
  try {
    const response = await fetch("https://pokeapi.co/api/v2/pokemon/ditto");
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error("Error fetching data:", error);
  }
}
fetchData();

4. JAVASCRIPT FATIGUE. FRAMEWORKS, EMPAQUETADORES Y TRANSPILADORES.

El mundo de los frameworks y librerías de JavaScript es extremadamente amplio y variado. React es uno de los más populares, pero no el único.

Tan abrumadora es la cantidad de herramientas, librerías, frameworks y tan constantes y veloces sus actualizaciones y cambios dentro del ecosistema, que se acuñó el término "Javascript Fatigue" para describir la sensación de abrumación que sienten muchos desarrolladores al intentar mantenerse al día con todas las novedades.

De cara al desarrollo web todo acaba traduciéndose en última instancia a HTML, CSS y JavaScript. Por eso, hasta ahora nos hemos centrado en aprenderlos, preparándonos para cualquiera que sea el framework popular del momento cuando nos toque trabajar con él.

En cualquier caso, para entender el papel de React (o cualquier otro framework) en el desarrollo web, es importante tener en cuenta el papel de otros elementos del ecosistema de desarrollo web, como los empaquetadores y los transpiladores.

4.1. Frameworks

Los frameworks son conjuntos de herramientas y librerías que nos permiten desarrollar aplicaciones web de forma más rápida y eficiente. React es un framework, pero no el único. Otros frameworks populares son Angular , Vue.js, Svelte, Next.js, Ember, Backbone, etc.

Echemos un breve vistazo a las estadísticas de algunos de ellos, para entender por qué nos decantamos por React. (Fuente: https://npmtrends.com/@angular/core-vs-angular-vs-backbone-vs-next-vs-react-vs-svelte-vs-vue)

4.2. Empaquetadores

Los empaquetadores son herramientas que nos permiten empaquetar y optimizar nuestro código JavaScript, CSS, imágenes, etc. para que sea más eficiente y rápido. Realizan funciones como la minimización de código, gestión de dependencias y optimización de carga. Algunos de los empaquetadores más populares son Webpack, Parcel, Vite, Rollup, Snowpack... Por defecto, con react, se utiliza Webpack. Aunque Vite está ganando mucha popularidad.

4.3. Transpiladores

Los transpiladores son herramientas que nos permiten transformar nuestro código JavaScript moderno en código JavaScript compatible con navegadores más antiguos. Algunos de los transpiladores más populares son Babel (utilizado por React) o SWC (https://swc.rs/) ...

5. REACT

React es una biblioteca de JavaScript para construir interfaces de usuario, desarrollada y mantenida por Facebook (ahora Meta). Pero vayamos directamente al grano!

  • El enfoque basado en componentes nos permite crear interfaces de usuario componiendo componentes más pequeños.

  • Actualiza y renderiza eficientemente los componentes cuyos datos han cambiado.

  • Las vistas declarativas hacen tu código más predecible y fácil debuguear.

5.1. Imperativo VS Declarativo

En lugar de manipular directamente el DOM o decirle al navegador cómo actualizar la interfaz de usuario (estilo imperativo), se declara cómo debería verse la UI en cualquier estado dado con JSX y componentes React, y el propio React se encarga de realizar las actualizaciones necesarias de manera eficiente (estilo declarativo).

5.2. Características

  • Librería, no framework: Es una biblioteca de JavaScript, no un framework completo; está diseñada para ser integrada con otras librerías y herramientas, no dicta la estructura completa de una aplicación.

  • Componentes en vez de plantillas: Utiliza componentes para construir interfaces de usuario, busca la modularidad y reutilización de código.

  • Simplifica el trabajo con el DOM: Maneja eficientemente el Document Object Model (DOM) mediante un DOM virtual, que permite actualizar solo las partes que necesitan ser re-renderizadas,

  • Enorme comunidad: Su popularidad hace que haya una gran cantidad de recursos, documentación y librerías de terceros disponibles.

  • Moderno: Utiliza JavaScript moderno, como ES6 y JSX, y se integra con otras herramientas modernas de desarrollo web como el transpilador babel o empaquetador webpack

  • React Native: Permite desarrollar aplicaciones móviles multiplataforma nativas con React.

  • Facebook, Instagram, Netflix, Apple, Uber, Amazon, PayPal, WhatsApp, Yahoo... hacen uso de React. Demuestra su escalabilidad, eficiencia, popularidad y versatilidad.

5.3. Client Side Rendering (CSR) VS Server Side Rendering (SSR)

React puede renderizar en el cliente o en el servidor. En el caso de renderizado en el cliente, el servidor envía al cliente un HTML vacío y React se encarga de renderizar la aplicación en el navegador. En el caso de renderizado en el servidor, el servidor envía al cliente un HTML con la aplicación ya renderizada. El renderizado en el servidor es más rápido y mejora el SEO, pero es más complicado de implementar. La tendencia actual es a utilizar el renderizado en el cliente, y así lo haremos en nuestros proyectos.

6.3.1. JSX

Cuando editamos el fichero App.js, veremos que estamos escribiendo algo que no es JavaScript puro. Eso es JSX (JavaScript XML), una extensión de JavaScript que nos permite escribir HTML en nuestro código JavaScript. JSX es una sintaxis que nos permite escribir código más limpio y fácil de leer.

Por ejemplo, el siguiente código javascript puro se encarga añadir al DOM un listado de frutas que proviene de un array:

var container = document.createElement("div");

var header = document.createElement("h1");
header.textContent = "Lista de Frutas";

var list = document.createElement("ul");
var fruits = ["Manzana", "Banana", "Cereza"];
fruits.forEach(function (fruit) {
  var li = document.createElement("li");
  li.textContent = fruit;
  list.appendChild(li);
});

container.appendChild(header);
container.appendChild(list);

document.body.appendChild(container);

Esta tarea se simplifica mucho con JSX:

const App = () => {
  {
    /* ⚠️ Presta atención a cómo cada vez que deseo evaluar o ejecutar código JavaScript dentro de JSX, lo hago entre llaves `{}`. Los comentarios también!*/
  }
  const fruits = ["Manzana", "Banana", "Cereza"];
  return (
    <div>
      <h1>Lista de Frutas</h1>
      <ul>
        {fruits.map((fruit) => (
          <li>{fruit}</li>
        ))}
      </ul>
    </div>
  );
};

Fíjate en cómo estamos entremezclando HTML y JavaScript. Esto es JSX. React se encargará de transformar este código en código JavaScript puro que el navegador pueda entender.

6.3.2. Componentes

Un componente es una pieza de código reutilizable que encapsula una parte de la interfaz de usuario. Los componentes pueden contener otros componentes, y así sucesivamente. Los componentes son la base de React.

React se va a encargar por su cuenta de vigilar el estado de todos los componentes y de actualizar solo aquellos que hayan cambiado. Esto es posible gracias a su algoritmo de reconciliación, que compara el árbol de componentes actual con el anterior y actualiza solo las partes que han cambiado.

8.2. Hooks

En React, un Hook es una función especial que te permite “enganchar” (hook) funcionalidades adicionales a los componentes. Los Hooks no funcionan dentro de clases, solo funcionan en componentes funcionales.

Los Hooks fueron introducidos en React 16.8 (Febrero 2019) y permiten usar el estado y otras características de React sin necesidad de escribir una clase para ello.

React integra multitud de Hooks y además podemos crear nuestros propios hooks. Los integrados más comunes son useState y useEffect. Vamos a verlos en detalle.

8.2.1. useState para manejo de estado

https://react.dev/reference/react/useState

El hook useState nos permite añadir estado a los componentes funcionales. El estado es una forma de almacenar datos que pueden cambiar a lo largo del tiempo. Cada vez que el estado cambia, React se encarga de re-renderizar el componente.

Partamos de un ejemplo sencillo. Imaginemos que queremos añadir un contador a nuestra aplicación. Con lo que sabemos de React y de Javascript hasta el momento, probablemente hiciéramos algo así:

function App() {
  let puntos = 50;
  const aumentarPuntos = () => {
    puntos = puntos + 1;
    console.log(puntos);
  };
  return (
    <div className="App">
      <p>Points: {puntos}</p>
      <button onClick={aumentarPuntos}>Sumar puntos</button>
    </div>
  );
}

Funciona, verdad? Lo has copiado y pegado en tu App.js y has comprobado que funciona? Pues no! No funciona! El contador no se actualiza en la pantalla. En la consola del navegador vemos que el contador está sumando y lo imprimimos en ella... Pero la interfaz no se da por enterada, y el contador de la interfaz se queda atascado en 50. Por qué? Porque React no sabe que el estado ha cambiado. React no sabe que debe re-renderizar el componente. Por eso necesitamos useState, para decirle a React que el estado ha cambiado y que debe re-renderizar el componente.

(A estas alturas ya deberías haberlo buscado, pero "renderizar" es solo una palabra chachi para decir "pintar en pantalla")

Al lio, solucionemos este asunto con useState:

// App.js
// Lo primero será importar useState para poder utilizarlo
import React, { useState } from "react";

function App() {
  // Y ahora utilizamos useState para crear una variable de estado y una función para modificarla, así como su valor inicial
  const [puntos, setPuntos] = useState(50);
  const aumentarPuntos = () => {
    setPuntos(puntos + 1);
  };
  return (
    <div className="App">
      <p>Points: {puntos}</p>
      <button onClick={aumentarPuntos}>Sumar puntos</button>
    </div>
  );
}

Okay, ahora sí! Ahora sí que funciona! Cada vez que hagamos click en el botón, el contador se incrementará en 1 y la interfaz se actualizará. Y todo gracias a useState.

Esta manera de utilizar el useState mediante una array de dos posiciones es un poco "extraña", repasemos qué está ocurriendo aquí.

  • En la primera posición del array, puntos es el valor actual del estado. Este valor se inicializa con el valor que le pasamos a useState como argumento. En este caso, 50. Esta variable es de solo lectura, no debemos modificarla directamente. Si queremos cambiarla, debemos utilizar la función que se nos devuelve en la segunda posición del array (setPuntos). Podemos llamar a estar variable como queramos, pero es una buena práctica que el nombre de la variable y de la función que la modifica sean descriptivos y se entiendan bien. Por ejemplo, si estamos manejando el estado de un contador, podríamos llamar a la variable contador y a la función setContador.
  • En la segunda posición del array, setPuntos es la función que nos permite modificar el estado. Digamos que es al función "setter" y que se encarga de notificar a React que el estado ha cambiado y que debe re-renderizar el componente. Esta función recibe un argumento, que es el nuevo valor del estado. En nuestro ejemplo recibe como arugmento puntos + 1.
  • Y por último, en la parte derecha de la asignación, useState recibe como argumento el valor inicial del estado. En nuestro caso, 50.

Ahora, cada vez que hagamos click en el botón, el estado puntos se incrementará en 1 y React se encargará de re-renderizar el componente con el nuevo valor del estado.

Vamos con otro ejemplo. Imaginemos que queremos añadir un input a nuestra aplicación para que el usuario pueda introducir su nombre. Nuestra aplicación debería ser capaz de recoger ese input en una variable, y de paso actualizar en tiempo real los textos que hacen referencia a ella. Vamos a utilizar useState para manejar el estado del input.

function App() {
  const [inputValue, setInputValue] = useState("");
  const handleInputChange = (event) => {
    setInputValue(event.target.value.toUpperCase());
  };
  return (
    <div className="App">
      <input
        type="text"
        placeholder="Escribe algo"
        onChange={handleInputChange}
      />
      <br />
      Has escrito: {inputValue}
    </div>
  );
}

Aprovechamos el evento onChange para llamar a la función handleInputChange cada vez que el usuario escriba algo en el input. Esta función recibe el evento como argumento y actualiza el estado inputValue con el valor del input en mayúsculas. De esta forma, cada vez que el usuario escriba algo en el input, el estado inputValue se actualizará y React se encargará de re-renderizar el componente con el nuevo valor del estado.

Por último, ten en cuenta que podemos hacer uso de useState, no solo para almacenar variables a dibujar en pantalla... También podemos utilizarlo para almacenar el estado de la aplicación, como por ejemplo, el estado de un modal, determinar si un elemento debe dibujarse o no o escoger el estilo de un elemnto...

Veamo un ejemplo en el que utilizamos useState para mostrar u ocultar un saludo al hacer click en un botón. Además nos va a servir como recordatorio de cómo utilizar el operador ternario.

function App() {
  const [mostrarSaludo, setMostrarSaludo] = useState(false);
  return (
    <div className="App">
      <button
        onClick={() => {
          setMostrarSaludo(!mostrarSaludo);
        }}
      >
        Show/hide
      </button>
      {mostrarSaludo && <h1>Hello world</h1>}
    </div>
  );
}

No es que el <h1>hello world</h1> no se vea, es que ni siquiera se ha renderizado en el DOM. Si inspeccionamos el código fuente de la página, veremos que no está. Y es que React no lo ha renderizado. Por qué? Porque el estado mostrarSaludo está inicializado a false, y el operador ternario que utilizamos para mostrar u ocultar el saludo solo muestra el saludo si mostrarSaludo es true. Por lo tanto, el saludo no se muestra. Pero si hacemos click en el botón, el estado mostrarSaludo cambia a true, y React se encarga de re-renderizar el componente con el nuevo valor del estado, mostrando el saludo.

Por último, veamos un ejemplo en el que utilizamos useState para cambiar el color de un texto al hacer click en un botón. Además nos va a servir como recordatorio de como aplicar estilos inline en React. Y además usamos funciones arrow anónimas para no tener que crear una función aparte.

function App() {
  const [textColor, setTextColor] = useState("black");
  return (
    <div className="App">
      <button
        onClick={() => {
          setTextColor(textColor === "black" ? "red" : "black");
        }}
      >
        Cambiar color
      </button>
      <h1 style={{ color: textColor }}>Hello world</h1>
    </div>
  );
}

Por qué estamos usando una función arrow anónima en el evento onClick? Porque queremos que la función que se ejecute al hacer click en el botón sea una función que no se va a reutilizar en ningún otro sitio. Si la función que se ejecuta al hacer click en el botón fuera una función que se va a reutilizar en otros sitios, entonces deberíamos crear una función aparte y llamarla en el evento onClick. Sin no usasemosa aquí una función anónima tendríamos que haber hecho algo así:

function App() {
  const [textColor, setTextColor] = useState("black");
  const cambiarColor = () => {
    setTextColor(textColor === "black" ? "red" : "black");
  };
  return (
    <div className="App">
      <button onClick={cambiarColor}>Cambiar color</button>
      <h1 style={{ color: textColor }}>Hello world</h1>
    </div>
  );
}

Un poco redundante, verdad? Por eso utilizamos una función arrow anónima en el evento onClick.

8.2.2. useEffect para manejo de efectos secundarios y ciclo de vida

https://react.dev/reference/react/useEffect

El hook useEffect nos permite realizar efectos secundarios en componentes funcionales. Los efectos secundarios son operaciones asíncronas, como llamadas a APIs, suscripciones a eventos, etc. useEffect se ejecuta después de cada renderizado del componente.

Okay, quizás ese párrafo anterior todavía no signifique nada para nosotros. Primero es importante que entendamos el ciclo de vida de un componente en React.

8.2.2.1. Ciclo de vida de un componente en React

En react, los componentes pasan por un ciclo de vida. Este ciclo de vida consta de 3 fases:

  • Montaje: El componente se monta en el DOM. En esta fase, se ejecuta el constructor, el método render y el método componentDidMount.
  • Actualización: El componente se actualiza en el DOM. En esta fase, se ejecuta el método render y el método componentDidUpdate.
  • Desmontaje: El componente se desmonta del DOM. En esta fase, se ejecuta el método componentWillUnmount.

Mejor pongamos en marcha un ejemplo:

// Intenta, solo observando el código, antes de ejecutarlo, entender qué va a ocurrir...
import "./App.css";
import { useState } from "react";
function App() {
  const [mostrarInput, setMostrarInput] = useState(false);
  return (
    <div className="App">
      <button
        onClick={() => {
          setMostrarInput(!mostrarInput);
        }}
      >
        {mostrarInput ? "Ocultar" : "Mostrar"} formulario
      </button>
      <br />
      {mostrarInput && <Formulario />}
    </div>
  );
}

function Formulario() {
  const [texto, setTexto] = useState("");
  return (
    <div>
      <input
        onChange={(event) => {
          setTexto(event.target.value);
        }}
        type="text"
      />
      <p>Has escrito: {texto}</p>
    </div>
  );
}

export default App;

Si lo ejecutamos y consultadmos el DOM desde el navegador, veremos como el componente Formulario realmente aparece y desaparece del DOM. Estamos viendo el ciclo de vida de un componente en acción. Pero... ¿Y si queremos hacer algo más en el ciclo de vida de un componente? ¿Y si queremos hacer algo cada vez que el componente se monta o se actualiza? Para eso utilizamos useEffect.

8.2.2.2. Empleo del useEffect

Continuamos con el ejemplo del punto anterior. Imaginemos que queremos mostrar un mensaje en la consola cada vez que el componente Formulario se monta o se actualiza. Para ello, utilizaremos useEffect.

//Debo  importar useEffect al igual que importabamos useState
import { useEffect, useState } from "react";

// componente App...

function Formulario() {
  const [texto, setTexto] = useState("");
  useEffect(() => {
    console.log("Componente montado!");
  });
  return (
    <div>
      <input
        onChange={(event) => {
          setTexto(event.target.value);
        }}
        type="text"
      />
      <p>Has escrito: {texto}</p>
    </div>
  );
}

Si ejecutamos el código, veremos que cada vez que el componente Formulario se monta o se actualiza, se muestra un mensaje en la consola.

Pero un momento... El mensaje en consola se está imprimiendo una vez de más, no? Esto es culpa/responsabilidad del StrictMode de React. Nuestro fichero index.js está envolviendo nuestra <App/> en un <React.StrictMode>. El <React.StrictMode> es un componente que nos ayuda a detectar problemas en nuestra aplicación. Uno de los problemas que detecta es que los efectos secundarios se están ejecutando dos veces. Esto es porque React ejecuta los efectos secundarios dos veces en modo de desarrollo para detectar posibles problemas. En producción, los efectos secundarios se ejecutarán solo una vez. Por lo tanto, no te preocupes si ves que los efectos secundarios se están ejecutando dos veces en modo de desarrollo. Puedes retirar el strict mode si te resulta molesto.

Entonces... ¿Por qué es útil esto del useEffect? Pues porque nos permite realizar operaciones asíncronas, como llamadas a APIs, suscripciones a eventos, etc. Imagina que tienes un componente que necesita hacer una llamada a una API para obtener datos. Puedes utilizar useEffect para realizar la llamada a la API cada vez que el componente se monta o se actualiza.

Pero, y si queremos que el efecto secundario se ejecute solo una vez, cuando el componente se monta, ignorando cuando se actualiza? Para ello, podemos pasar un array vacío como segundo argumento de useEffect. Este array se conoce como array de dependencias. Si el array de dependencias está vacío, el efecto secundario se ejecutará solo una vez, cuando el componente se monte. Si el array de dependencias no está vacío, el efecto secundario se ejecutará cada vez que el componente se monte y cada vez que una de las dependencias cambie.

En el siguiente ejemplo el useEffect se ejecutará al montarse y cada vez que cambie el texto. Si queremos que se ejecute solo al montarse, debemos pasar un array vacío como segundo argumento.

import { useEffect, useState } from "react";

// componente App...

function Formulario() {
  const [texto, setTexto] = useState("");
  useEffect(() => {
    console.log("Componente montado!");
  }, [texto]);
  return (
    <div>
      <input
        onChange={(event) => {
          setTexto(event.target.value);
        }}
        type="text"
      />
      <p>Has escrito: {texto}</p>
    </div>
  );
}

Para ejecutar algo al desmontar el componente, podemos devolver una función en el useEffect. Esta función se ejecutará cuando el componente se desmonte. Esto puede ser interesante por ejemplo para limpiar suscripciones a eventos, cancelar peticiones incompletas a APIs, etc.

Un ejemplo que reacciona a todo el ciclo de vida:

function Formulario() {
  const [texto, setTexto] = useState("");
  useEffect(() => {
    console.log("Componente Montado");
    return () => {
      console.log("Componente Desmontado");
    };
  }, []);
  useEffect(() => {
    console.log("Componente Actualizado");
  }, [texto]);

  return (
    <div>
      <input
        onChange={(event) => {
          setTexto(event.target.value);
        }}
        type="text"
      />
      <p>Has escrito: {texto}</p>
    </div>
  );
}

Ejecútalo y comprueba en la consola que se muestra el mensaje "Componente Montado" al montar el componente, "Componente Actualizado" al actualizarlo y "Componente Desmontado" al desmontarlo.

Por último, esto del useEffect puede sernos muy util también para, por ejemplo, vincular algunos ficheros javascript a ciertos componentes...

useEffect(() => {
  const script = document.createElement("script");

  script.src = "urlDeMiScript";
  script.async = true;
  // Insertamos el script en el DOM al montar el componente
  document.body.appendChild(script);

  // Eliminamos el script del DOM al desmontar el componente
  return () => {
    document.body.removeChild(script);
  };
}, []);

9.2. Consumir una API

Hasta el momento, los datos de nuestra aplicación los hemos almacenado en variables o en arrays dentro de nuestros componentes. Pero en la vida real, los datos de nuestra aplicación suelen estar almacenados en una base de datos o en un servidor. Para acceder a estos datos, necesitamos hacer una petición a una API.

Generalmente, nuestras aplicaciones se comunican con una API REST. REST es un estilo de arquitectura que define un conjunto de restricciones para el diseño de servicios web. REST se basa en el protocolo HTTP y utiliza los métodos de HTTP para realizar operaciones CRUD (Create, Read, Update, Delete) sobre los recursos.

Para nuestros proyectos nos hemos repartido un conjunto de APIs con distintas temáticas y características. No todas las APIs son iguales ni devuelven información con la misma estructura, pero todas ellas se comunican con nuestra aplicación a través de peticiones HTTP.

9.2.1 fetch()

https://developer.mozilla.org/en-US/docs/Web/API/fetch

Fetch es una función nativa de Javascript que nos permite hacer peticiones HTTP a una API.

Prueba a ejecutar el siguiente código en la consola del navegador:

fetch("https://api.open5e.com/v1/monsters/");

Como ves, fetch devuelve una promesa que resuelve en un objeto Response. Para obtener los datos de la respuesta, debemos esperar a que se resuelva dicha promesa... Prueba este código en la consola del navegador:

fetch("https://api.open5e.com/v1/monsters/").then((response) => {
  console.log(response);
});

Eso ya tiene mejor aspecto, verdad? Pero... ¿Dónde están los datos? ¿Cómo obtenemos los datos de la respuesta? Para ello, debemos llamar al método json() del objeto Response. Prueba este código en la consola del navegador:

fetch("https://api.open5e.com/v1/monsters/").then((response) => {
  response.json().then((data) => {
    console.log(data);
  });
});

Ahora sí que sí, la API nos ha devuelto los datos y los hemos mostrado en la consola.

De hecho esta API en concreto nos ha devuelto una respuesta muy bien organizada, con un contador de monstruos total, un enlace a la página siguiente y un array con los datos de los monstruos. Pero no todas las APIs son tan amables... Algunas APIs devuelven datos en bruto, sin estructurar, y es nuestra responsabilidad organizarlos y mostrarlos en nuestra aplicación.

9.2.2. fetch() en React

Tras haber probado fetch en la consola del navegador, vamos a utilizarlo en nuestra aplicación React. Vamos a hacer una petición a la API de Open5e para obtener los datos de los monstruos y mostrarlos en nuestra aplicación.

Ya sabemos que el useEffect es el lugar ideal para realizar efectos secundarios en un componente. Por lo tanto, vamos a utilizar useEffect para hacer la petición a la API y almacenar los datos en el estado del componente. De esta manera no realizaremos la petición a la API hasta que sea necesario, es decir, hasta que el componente se monte.

Supongamos que tenemos una App que muestra una lista de monstruos. Probablemente tengamos un componente ListaMonstruos que se encargue de mostrar la lista de monstruos. Vamos a hacer la petición a la API en el componente ListaMonstruos. Pero antes, un botón en el componente App para mostrar u ocultar la lista de monstruos.

// App.js
import "./App.css";
import { useState } from "react";
import ListaMonstruos from "./components/ListaMonstruos";
function App() {
  const [mostrarMonstruos, setMostrarMonstruos] = useState(false);
  const toggleMonstruos = () => {
    setMostrarMonstruos(!mostrarMonstruos);
  };
  return (
    <div className="App">
      <h1>Enciclopedia D&D</h1>
      <button onClick={toggleMonstruos}>Ocultar monstruos</button>
      {mostrarMonstruos ? <ListaMonstruos /> : ""}
    </div>
  );
}

export default App;
// components/ListaMonstruos.js
import { useEffect } from "react";
function ListaMonstruos() {
  useEffect(() => {
    console.log("Montado componente ListaMonstruos");
    fetch("https://api.open5e.com/v1/monsters/").then((response) => {
      response.json().then((data) => {
        console.log(data);
      });
    });
  }, []);
  return (
    <div>
      <h2>Lista de monstruos</h2>
    </div>
  );
}
export default ListaMonstruos;

Como podrás comprobar, al hacer click en el botón "Ocultar monstruos" se monta el componente ListaMonstruos y se realiza la petición a la API. Los datos de los monstruos se muestran en la consola. (Todavía no estamos mostrando los datos en la aplicación, pero eso lo haremos en un momento). Ahora fíjate como gracias al useEffect, la petición a la API se realiza cada vez que el componente se monta.

9.2.3. Sustituimos fetch por axios

https://www.npmjs.com/package/axios

¿Por qué hacemos esto? Porque axios, a pesar no ser nativo de Javascript, es más sencillo de utilizar y más potente que fetch. Además, axios nos permite manejar errores de una manera más sencilla. Más de 50 millones de descargas semanales lo avalan...

Para utilizar axios, primero debemos instalarlo en nuestro proyecto. Para ello, abrimos una terminal y ejecutamos el siguiente comando:

npm install axios

Una vez instalado, podemos importarlo en nuestro componente y utilizarlo para hacer la petición a la API. Vamos a sustituir fetch por axios en nuestro componente ListaMonstruos.

// components/ListaMonstruos.js
import { useEffect } from "react";
import Axios from "axios";
function ListaMonstruos() {
  useEffect(() => {
    console.log("Montado componente ListaMonstruos");

      console.log(response.data.results);
    });

      <h2>Lista de monstruos</h2>
    </div>
  );
}

Comprueba que efectivamente todo sigue funcionando y los datos de los monstruos se imprimen en consola.

Obviamente nos falta renderizarlos en la aplicación, para lo cual necesitamos almacenar los datos en el estado del componente. Menos mal que ya controlamos el useState...

import { useEffect, useState } from "react";
import Axios from "axios";
function ListaMonstruos() {

  return (
    <div>
      <h2>Lista de monstruos</h2>
      {monstruos.map((monstruo) => (
        <div key={monstruo.index}>
          {monstruo.name} - {monstruo.size} - {monstruo.type} -{" "}
          {monstruo.alignment}
        </div>
      ))}
    </div>
  );
}
export default ListaMonstruos;

Ahora sí, los datos de los monstruos se muestran en la aplicación. Hemos hecho una petición a la API, almacenado los datos en el estado del componente y mostrado los datos en la aplicación. 9.2.5. Paginación en la API

Por suerte, la API de Open5e nos permite obtener los datos de los monstruos paginados. Si consultamos la API en el navegador, veremos que la respuesta contiene un contador de monstruos total, un enlace a la página siguiente y un array con los datos de los monstruos. Si queremos obtener los datos de la página siguiente, debemos hacer una petición a la URL que aparece en el campo next de la respuesta.

Podemos aprovechar esta funcionalidad para implementar una paginación en nuestra aplicación. Haremos uso del useState para llevar un contador de la página actual, y de un botón para avanzar a la página siguiente. Seguiremos usando useState también para almacenar la información de los monstruos de la página actual. Y además usaremos useEffect para hacer la petición a la API y actualizar esa información cada vez que cambie el número de página.

import { useEffect, useState } from "react";
import Axios from "axios";
import Monstruo from "./Monstruo";
function ListaMonstruos() {
  const [monstruos, setMonstruos] = useState([]);
  const [pagina, setPagina] = useState(1);
  useEffect(() => {
    console.log("Montado componente ListaMonstruos");
    Axios.get(`https://api.open5e.com/v1/monsters/?page=${pagina}`).then(
      (response) => {
        console.log(response.data.results);
        setMonstruos(response.data.results);
      }
    );
  }, [pagina]);

  return (
    <div>
      <h2>Lista de monstruos</h2>
      <p>Estas en la página {pagina}.</p>
      <button
        onClick={() => {
          setPagina(pagina + 1);
        }}
      >
        Pasar a página {pagina + 1}
      </button>
      {monstruos.map((monstruo, key) => (
        <Monstruo monstruo={monstruo} key={key} />
      ))}
    </div>
  );
}
export default ListaMonstruos;

Evolucionar esto para poder retroceder y que el botón de "siguiente página" esté deshabilitado cuando no haya más páginas por delante, y que no se pueda retroceder a una página anterior a la primera. Deberías se capaz de hacerlo después de lo aprendido hasta ahora... Pero mejor si invertimos este tiempo en aplicarlo a tu proyecto final.

9.2.6. Búsqueda en la API

Algunas APIS, como afortunadamente es el caso de la API de Open5e, nos permiten realizar búsquedas. En este caso, podemos hacer una petición a la API con un término de búsqueda y obtener los datos que coincidan con ese término. En su documentación open5e nos indica que podemos hacer una búsqueda en el endpoint /search/ con el parámetro text. Vamos a implementar una barra de búsqueda en nuestra aplicación para buscar información...

import "./App.css";
import { useState } from "react";
import ListaMonstruos from "./components/ListaMonstruos";
import Axios from "axios";
function App() {
  const [mostrarMonstruos, setMostrarMonstruos] = useState(false);
  const [terminoBusqueda, setTerminoBusqueda] = useState("");
  const [resultadosBusqueda, setResultadosBusqueda] = useState([]);
  const toggleMonstruos = () => {
    setMostrarMonstruos(!mostrarMonstruos);
  };
  const buscarTermino = () => {
    Axios.get(`https://api.open5e.com/search/?text=${terminoBusqueda}`).then(
      (response) => {
        setResultadosBusqueda(response.data.results);
      }
    );
  };
  return (
    <div className="App">
      <section className="searchContainer">
        <input
          type="text"
          placeholder="Busca algo aqui!"
          onChange={(event) => {
            setTerminoBusqueda(event.target.value);
          }}
        />
        <button onClick={buscarTermino}>Buscar</button>
        <br />
        <table>
          <tbody>
            {resultadosBusqueda.map((resultado, key) => (
              <tr key={key}>
                <td style={{ border: "1px solid black" }}>{resultado.text}</td>
              </tr>
            ))}
          </tbody>
        </table>
      </section>
      <h1>Enciclopedia D&D</h1>
      <button onClick={toggleMonstruos}>Ocultar monstruos</button>
      {mostrarMonstruos ? <ListaMonstruos /> : ""}
    </div>
  );
}
export default App;

9.3. Rutas en React

https://reactrouter.com/en/main/start/tutorial

Hemos aprendido multitud de cosas interesantes en React, pero hasta ahora solo hemos trabajado con una única página. Todavía nos falta algo ESENCIAL. En la vida real, nuestras aplicaciones suelen tener varias páginas, o al menos varias rutas. Para manejar las rutas en React, podemos utilizar la librería react-router-dom.

Una vez más, tenemos que instalar la librería en nuestro proyecto. Para ello, abrimos una terminal y ejecutamos el siguiente comando:

npm install react-router-dom

Y comprobamos que efectivamente esta nueva dependencia se ha añadido al fichero package.json.

Para utilizar react-router-dom, primero debemos importar los componentes BrowserRouter, Routes (as Router) y Route en nuestro componente principal App.

  • BrowserRouter (que llamaremos simplemente Route) es el componente que envolverá nuestra aplicación y nos permite manejar las rutas. Todo el contenido de nuestra aplicacion que debe reaccionar a las rutas debe estar dentro de un componente BrowserRouter.
  • Routes es el componente que contiene todas las rutas de nuestra aplicación. En su interior, debemos añadir un componente Route por cada ruta de nuestra aplicación.
  • Route es el componente que renderiza un componente cuando la ruta coincide con la URL actual. Cada Route debe tener dos propiedades: path y element. path es la ruta que debe coincidir con la URL actual, y element es el componente que se renderizará cuando la ruta coincida con la URL actual.

Puede ser lioso por escrito, pero es muy sencillo de implementar. Vamos a verlo en acción...

// App.js
import "./App.css";

import { BrowserRouter as Router, Routes, Route } from "react-router-dom";
function App() {
  return (
    <div className="App">
      <Router>
        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/about" element={<About />} />
          <Route path="/contact" element={<Contact />} />
        </Routes>
      </Router>
    </div>
  );
}

function Home() {
  return <h1>Estás en la sección Home</h1>;
}

function About() {
  return <h1>Estás en la sección About</h1>;
}

function Contact() {
  return <h1>Estás en la sección Contact</h1>;
}

export default App;

Fíjate en cómo cada "página" de mi SPA es simplemente un componente (que puede contener muchos otros...). Y fíjate en cómo cada componente se renderiza cuando la URL coincide con la ruta indicada en el componente Route.

Si además quieres definir una página de error 404, puedes hacerlo añadiendo una ruta con el path * al final de todas las rutas. De esta forma, si ninguna de las rutas anteriores coincide con la URL actual, se renderizará la página de error 404.

<Route path="*" element={<h1>Not Found</h1>} />

En principio con esto ya nos bastaría. Podrías colocar enlaces tradicionales como:

<a href="/about">About</a>

Y funcionarían perfectamente. Pero si quieres hacerlo más bonito, puedes utilizar el componente Link de react-router-dom. Link es un componente que nos permite navegar entre rutas sin recargar la página. Para utilizar Link, simplemente debemos importarlo y utilizarlo en lugar de un enlace tradicional. Conviene que lo hagas en lugar de los enlaces tradicionales, ya que los enlaces tradicionales recargan la página y pierden el estado de la aplicación.

// App.js
import { BrowserRouter as Router, Routes, Route, Link } from "react-router-dom";
function App() {
  return (
    <div className="App">
      <Router>
        <nav>
          <ul>
            <li>
              <Link to="/">Home</Link>
            </li>
            <li>
              <Link to="/about">About</Link>
            </li>
            <li>
              <Link to="/contact">Contact</Link>
            </li>
          </ul>
        </nav>
        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/about" element={<About />} />
          <Route path="/contact" element={<Contact />} />
          <Route path="*" element={<h1>Not Found</h1>} />
        </Routes>
      </Router>
    </div>
  );
}

Fíjate en como estamos dejando dentro de <Router> no solo las rutas, sino también un menú de navegación. Esto es porque todo lo que queramos que reaccione a las rutas debe estar dentro de un componente BrowserRouter. Y fíjate en cómo estamos utilizando el componente Link en lugar de un enlace tradicional. De esta forma, la navegación entre rutas se realiza sin recargar la página. Dentro de <Routes> es donde se acaban dibujando los componentes que corresponden a cada ruta.

10. Fuera del scope de la asignatura

Nuestro tiempo es limitado y creo que lo hemos exprimido al máximo. Pero si te interesa seguir aprendiendo sobre React, aquí tienes algunas sugerencias sobre qué puede merecer la pena aprender a continuación:

  • Context API: Nos permite compartir datos entre componentes sin tener que pasar props manualmente a través de cada nivel de la jerarquía de componentes. La Context API es especialmente útil cuando tenemos datos que deben ser accesibles por muchos componentes a diferentes niveles de la jerarquía. https://react.dev/learn/passing-data-deeply-with-context#before-you-use-context

  • Hooks avanzados: Además de useState y useEffect, React nos proporciona otros hooks que pueden ser útiles en ciertas situaciones. Algunos de estos hooks son useReducer, useContext, useMemo, useCallback, useRef etc.

  • Formularios: Podemos utilizar librerías como Formik o React Hook Form para manejar formularios de una manera más avanzada, validar los datos, etc.

  • ReactQuery: ReactQuery (ahora en realidad se llama TanStack Query) se encarga de manejar la caché de los datos, las peticiones a la API, la gestión de errores, etc. A diferencia de axios, que es una librería para hacer peticiones a una API, ReactQuery es una librería para manejar los datos de la API en nuestra aplicación. Es decir que axios y ReactQuery no son excluyentes, sino que se complementan. https://tanstack.com/query/latest/docs/framework/react/overview

  • Custom Hooks: Los custom hooks son una forma de reutilizar lógica entre componentes. Los custom hooks son funciones que contienen lógica común y que pueden ser utilizadas en varios componentes. https://react.dev/learn/reusing-logic-with-custom-hooks

  • Suspense y Lazy: Suspense y Lazy son dos características de React que nos permiten cargar componentes de manera lazy. Que un componente se importe solo cuando es necesario, o mostrar animaciones de carga mientras se carga un componente entre otras... https://react.dev/reference/react/Suspense y https://react.dev/reference/react/lazy

  • Testing: Jest y React Testing Library para realizar pruebas unitarias y de integración.

  • Server Side Rendering: Server Side Rendering (SSR) es una técnica que nos permite renderizar la aplicación en el servidor y enviar el HTML generado al cliente. SSR mejora el rendimiento de la aplicación y la indexación en los motores de búsqueda.

  • React Native: React Native es un framework que nos permite desarrollar aplicaciones móviles para iOS y Android utilizando React. https://reactnative.dev/

  • TypeScript: TypeScript es un superconjunto de JavaScript que añade tipado estático a JavaScript. TypeScript nos ayuda a detectar errores en tiempo de compilación y a escribir código más seguro y mantenible. https://www.typescriptlang.org/

En realidad, la documentación oficial de React es una fuente inagotable de información. Está excepcionalmente bien organizada y repleta de ejemplos. Siempre puedes consultar la documentación oficial de React para aprender más sobre React y sus características.
